# Lab 04 - Inventory Formats

## Objectives

- Write the same inventory in **INI** and **YAML** formats.
- Assign variables at host and group level.
- Understand when each format is preferable.
- Get a first look at dynamic inventories.

## Background

Ansible supports several inventory formats. The two static ones you will use most are **INI**
and **YAML**. They are equivalent in capability; YAML scales better for complex variable
structures, while INI is terser for simple host lists.

## Steps

### 1. INI format (review + variables)

Create `inventory.ini`:

```ini
[webservers]
web01 ansible_host=192.168.56.11
web02 ansible_host=192.168.56.12

[webservers:vars]
http_port=80
app_env=staging

[dbservers]
db01 ansible_host=192.168.56.21
```

`[group:vars]` assigns variables to every host in a group.

### 2. The same inventory in YAML

Create `inventory.yml`:

```yaml
---
all:
  children:
    webservers:
      hosts:
        web01:
          ansible_host: 192.168.56.11
        web02:
          ansible_host: 192.168.56.12
      vars:
        http_port: 80
        app_env: staging
    dbservers:
      hosts:
        db01:
          ansible_host: 192.168.56.21
```

Key structural words: `all`, `children`, `hosts`, `vars`.

### 3. Prove they are equivalent

```bash
ansible-inventory -i inventory.ini --graph --vars
ansible-inventory -i inventory.yml --graph --vars
```

Both should show the same hosts, groups, and variables.

### 4. Host ranges (INI shorthand)

INI supports numeric and alphabetic ranges - very handy for fleets:

```ini
[workers]
worker[01:05].example.com      # worker01 ... worker05
db-[a:c].example.com           # db-a, db-b, db-c
```

### 5. Separate variable files (recommended)

Rather than cramming variables into the inventory, Ansible auto-loads them from
`group_vars/` and `host_vars/` next to the inventory:

```bash
mkdir -p group_vars host_vars
```

`group_vars/webservers.yml`:

```yaml
---
http_port: 80
app_env: staging
```

`host_vars/web01.yml`:

```yaml
---
http_port: 8080 # overrides the group value for web01 only
```

Verify the override:

```bash
ansible-inventory -i inventory.yml --host web01
```

`http_port` for `web01` should be `8080`.

### 6. A peek at dynamic inventory

For cloud environments the inventory can be generated by a plugin or script instead of a static file. You point `-i` at an executable script or a `*.aws_ec2.yml` plugin config, and Ansible calls it to fetch the current host list. You will not configure one in this beginner course, but know that `-i` accepts directories and executables, not just static files.

## Choosing a format

| Use INI when...                 | Use YAML when...                    |
| ------------------------------- | ----------------------------------- |
| The host list is short and flat | You have nested/complex variables   |
| You want minimal typing         | You want consistency with playbooks |
| Quick lab/test inventories      | Larger, structured environments     |

## Verification

- The INI and YAML inventories produce identical `--graph --vars` output.
- `host_vars/web01.yml` successfully overrides the group's `http_port`.

## Challenge

Convert this INI snippet to YAML and confirm with `ansible-inventory --graph`:

```ini
[eu]
eu1 ansible_host=10.0.1.1
eu2 ansible_host=10.0.1.2
[eu:vars]
region=europe
```

## Key takeaways

- INI and YAML inventories are interchangeable; pick per situation.
- `group_vars/` and `host_vars/` keep variables out of the inventory file.
- More specific scope (host) wins over less specific (group).
